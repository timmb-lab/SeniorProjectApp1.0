# Administrator FAQ

## Is this replacing Canvas?

No. Not claimed. The demo positions Capstone Project as an operations layer for capstone visibility, review, mentor coverage, and readiness. Canvas or another LMS can remain the instructional home.

## Is this replacing Remind or school messaging?

No. Not claimed. Announcements and student messaging are intentionally removed from the active MVP surface. Schools should continue using district-approved communication tools.

## Is this using real student data?

No. Fake-data demo only. The local demo uses synthetic `.test` users and fake Desert Valley School District records.

## Is this FERPA compliant today?

Not claimed. The MVP is designed around role-scoped visibility, private evidence language, redaction, and audit-sensitive operations, but the demo is not a FERPA certification or legal approval.

## Who can see student evidence?

Proven locally: access is role-scoped. Administration can see selected-site operational detail, program teachers see scoped student/review data, mentors see assigned-student support data, and students remain on self-service paths. Raw Drive/storage identifiers are redacted from the site demo APIs.

## Can principals or APs see the whole school?

Proven locally for the fake site Administration role: assigned site administrators can view selected-site dashboard, directory, student detail, mentor assignments, review queue read-only, and operations readiness.

## Can teachers only see their assigned or scoped students?

Proven locally: program teachers are scoped to their selected-site program/cohort visibility. In the Review Queue they can mutate only in-scope submitted work.

## Can mentors only see assigned students?

Proven locally: mentors remain assigned-student only through mentor dashboard/detail support paths and are denied from site operations management routes.

## Can schools export or retain their data?

Planned / future. Archive readiness and export metadata are visible in the fake-data demo, and existing archive/export routes are preserved, but tenant-owned storage, retention policy, and production export operations need a later technical/legal gate.

## What is needed before a real pilot?

Hosted proof, remote migration 0011, remote fake-data seed/proof, SSO policy, legal/security review, data ownership decisions, district approval, support process, and a no-secret/no-real-data readiness check.

## What is currently local-only versus hosted?

Proven locally: the multisite demo routes and workspace screens. Hosted fake-data API/data proof is also ready after Phase 13C. Phase 14 added hosted browser screenshots with caveats: existing fake hosted admin and program teacher paths work, but generated remote staff credentials and viewer browser proof remain blocked.

## What happens if a school stops using it?

Planned / future. Offboarding, retention, tenant-owned storage, and export policy need to be specified before real data onboarding.

## Where would evidence live long-term?

Planned / future. Current demo evidence uses safe fake metadata and `https://example.com/capstone-demo/...` links. Tenant-owned Drive or other storage policy is a future decision unless separately proven.

## Can this work for multiple schools?

Proven locally with fake data: one fake district has three fake sites, with selected-site scoping and proof that the primary site does not leak secondary-site students.

## Can students or parents message inside it?

No. Not claimed. Student messaging and announcements are not part of the active MVP demo.
