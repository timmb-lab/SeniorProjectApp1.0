# Demo Screenshot Checklist

Screenshots were not generated in Phase 13. This file is only a checklist for a later manual or scripted capture pass.

Each screenshot must be labeled with date, environment, persona, and claim status. Do not capture passwords, `.secrets`, credential files, tokens, OAuth secrets, raw Drive IDs, private URLs, or raw D1 tables.

## Targets

- [ ] Sign-in / product header.
- [ ] Site Dashboard as Avery Administration.
- [ ] Student Directory with a story filter.
- [ ] Student Detail / Timeline for `Rich Timeline Demo`.
- [ ] Review Queue as Primary IT Program Teacher.
- [ ] Mentor Assignments as Avery Administration.
- [ ] Operations Readiness with archive failed or presentation pending filter.
- [ ] Viewer read-only state as Valeria Viewer.
- [ ] Permission denied/read-only states.
- [ ] Mobile or narrow viewport if supported by the demo plan.

## Required Label Format

Use:

```text
YYYY-MM-DD - Environment - Persona - Screen - Claim label
```

Example:

```text
2026-05-24 - local fake-data demo - Avery Administration - Site Dashboard - Proven locally
```
