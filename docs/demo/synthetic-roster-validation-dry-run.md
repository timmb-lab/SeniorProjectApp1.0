# Synthetic Roster Validation Dry Run

Status: **SYNTHETIC DRY-RUN CHECKLIST READY FOR REVIEW; real roster validation still MANUAL_PROOF_REQUIRED.**

This document maps roster validation to the current import UI/API contract using fake `.test` rows only. It does not replace `docs/progress/runs/real-student-pilot-roster-validation-evidence.json`.

## Evidence Boundary

- Use fake names and `.test` emails only.
- Do not paste real roster rows into docs, issues, screenshots, prompts, or proof artifacts.
- Do not perform real student import until the roster owner and privacy/support owner approve the source, fields, credential path, and support process.
- Treat CSV preview as synthetic validation evidence only. It does not prove the real roster is correct.

## Supported Student CSV Contract

Student CSV rows use the template generated by `workspace.js` and submitted through `/api/admin/users/import`.

Required columns:

- `first_name`
- `last_name`
- `email`
- `site`
- `program`

Optional columns:

- `cohort`
- `graduation_year`
- `status`
- `mentor_email`
- `viewer_email`

Important validations:

- Unsupported columns are blocked.
- Duplicate columns are blocked.
- Duplicate emails in the same CSV are blocked.
- Existing account emails are detected before import.
- `site` and `program` must be in the current admin scope.
- `graduation_year` must be a four-digit year when present.
- `mentor_email` and `viewer_email` must already exist in the current roster.
- Student users cannot be assigned as mentors or viewers.
- Final import is blocked if roster profile storage is unavailable with `student_roster_profiles_migration_required`.
- Real local account creation outside approved local-only settings is blocked with `credential_delivery_policy_required`.

## Supported Staff CSV Contract

Required columns:

- `first_name`
- `last_name`
- `email`
- `role`

Optional columns:

- `site`
- `program`
- `assigned_student_emails`
- `status`

Important validations:

- CSV import cannot create Global Admin accounts.
- Student rows must use Import Students, not Staff CSV.
- Site Admin and Administration need a site.
- Program Teacher needs a program.
- Mentor and Viewer need a site or assigned students.
- Assigned student emails must be inside the current admin scope.

## Synthetic Rows To Rehearse

| Case | Example shape | Expected result |
| --- | --- | --- |
| Valid student | `.test` student with site, program, cohort, graduation year, mentor email, viewer email | Preview valid; assignment counts show mentor/viewer if staff exists |

| Invalid graduation year | `26` or `Class of 2026` | Row error: `graduation_year` must be four digits |
| Unknown site/program | site or program outside current scope | Row error: site/program is not in current scope |
| Unknown mentor/viewer | mentor/viewer email not in current roster | Row error: mentor/viewer email must already exist |
| Student as mentor/viewer | student email in mentor/viewer column | Row error: student users cannot be assigned as mentor/viewer |
| Staff Global Admin | staff role `global_admin` | Row error: CSV import cannot create Global Admin accounts |
| Mentor without scope | mentor row with no site and no assigned students | Row error: Mentor and Viewer rows need a site or assigned students |
| Program Teacher without program | staff row role `program_teacher` and blank program | Row error: Program Teacher rows need a program |

## Dry-Run Procedure

1. Confirm the current user is an approved fake Global Admin, Site Admin, Administration, or Program Teacher.
2. Open Admin Console, then Imports.
3. Download or copy the Student CSV template.
4. Paste the synthetic rows above.
5. Click Preview.
6. Confirm each expected error or valid row appears before final import.
7. Do not click final import with real rows.
8. If using fake rows for local proof, keep generated credentials only in ignored `.secrets/` files and never commit them.
9. Run the focused import tests and readiness preflight.
