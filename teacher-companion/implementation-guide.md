# Senior Capstone Teacher Companion

This file is not linked from the student guide. Use it as a planning companion for teachers, mentors, and program teams.

## Instructional Sequence

1. Purpose and program requirements
   - Mini-lesson: why the Senior Capstone exists.
   - Student task: complete Program Requirement Notes.
   - Evidence: two project ideas with CTE connection and feasibility notes.

2. Core Concept Proposal
   - Mini-lesson: turning an interest into a feasible project.
   - Student task: draft, peer review, revise, submit.
   - Evidence: approved proposal and rubric self-check.

3. Build and check-ins
   - Mini-lesson: work sessions, artifacts, and honest progress reflection.
   - Student task: collect weekly evidence and complete check-ins.
   - Evidence: build plan, artifacts, mentor notes, revision log.

4. Presentation preparation
   - Mini-lesson: project storytelling and audience clarity.
   - Student task: presentation outline, practice, mentor feedback.
   - Evidence: outline, slides or visuals, practice feedback.

5. Celebration Day display
   - Mini-lesson: making process and product visible.
   - Student task: display plan, visitor test, setup.
   - Evidence: display photos, labels, visitor questions.

6. Gratitude, portfolio, and final reflection
   - Mini-lesson: professional evidence and reflection beyond summary.
   - Student task: thank-you letter, handwritten note, portfolio path, final reflection.
   - Evidence: portfolio submission or folder, resume, reflections, gratitude evidence.

## Formative Intervention Flags

- Student cannot name a CTE skill connected to the idea.
- Proposal describes a topic but not a product, process, service, research, or demonstration.
- Student has no visible artifact by the first build check-in.
- Mentor meeting notes do not include feedback or next actions.
- Presentation outline is a list of slides instead of a story of the work.
- Display is decorative but does not show process or product.
- Portfolio reflections summarize events without explaining growth.

## Differentiation Moves

- If stuck: reduce the task to the next visible artifact.
- If emerging: provide sentence frames and weak/better/strongest examples.
- If on track: require peer feedback and one revision.
- If advanced: push for testing, community impact, professional documentation, or recognition evidence.

## Rubric Calibration Prompts

- What does mastery evidence look like in this pathway?
- What would make this project too small for a senior-level capstone?
- Which artifacts show process, not just final product?
- What evidence would convince a visitor who does not know the student?
- Which students may qualify for service recognition or distinction?

## Mentor Guidance

Mentors should ask questions, review progress, and help students think clearly. They should not complete the project work.

Useful mentor questions:

- What problem, audience, or goal is this project addressing?
- What evidence proves progress since the proposal?
- What has changed because of feedback or testing?
- What part of the final presentation may confuse an audience?
- What should the student save for the portfolio?

## Suggested Class Routines

- Five-minute weekly artifact check.
- Peer read of project overview before proposal submission.
- Mentor-meeting prep day with three required questions.
- Presentation outline rehearsal with one peer note.
- Display visitor test before Celebration Day.
- Portfolio evidence sort before final submission.
